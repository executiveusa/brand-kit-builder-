## 13. STEVE KRUG USABILITY LAWS

Every website, interface, artifact, and digital brand application must pass these tests:

- A first-time visitor understands what it is, who it serves, and what to do within five seconds.
- One dominant action appears above the fold.
- Each screen has one primary workflow.
- Users can scan headings, labels, and grouped content.
- Visual hierarchy matches actual priority.
- Clickable elements look clickable without hover.
- Navigation uses familiar patterns unless a tested alternative is better.
- Page names and destinations match what users clicked.
- The logo returns home where expected.
- Users know where they are and how to leave modals, forms, and dead ends.
- Forms ask only for necessary information.
- Errors state what happened and how to recover.
- Mobile interactions are comfortable and one-handed where appropriate.
- Trust evidence is truthful and placed near decisions.
- Remove words, decoration, and motion that do not help the user understand or act.
- Delight may never weaken clarity, accessibility, speed, or conversion.

Run a trunk test from every major page.

## 14. ANTI-SLOP DESIGN PROTOCOL

Reject these unless a documented product reason overrides the ban:

- generic AI hero copy;
- fake metrics, dashboards, charts, testimonials, reviews, partnerships, urgency, or backend states;
- floating card shells;
- cards nested inside cards;
- oversized radii used everywhere;
- pill overload;
- decorative eyebrow labels;
- random purple/blue gradients;
- glass everywhere;
- random glows, blur haze, neon, or frosted panels;
- generic dark SaaS composition;
- repeated three-column icon-card sections;
- metric-card grids as a first instinct;
- meaningless bento grids;
- stock-photo filler;
- repeated assets used to fill space;
- overpadded layouts and dead space pretending to be luxury;
- centered generic hero text with no product reason;
- custom cursors that harm accessibility or performance;
- over-animation, scroll hijacking, forced intros, and distracting loops;
- decorative copy that could fit any business;
- template names, filler data, and placeholder identities;
- UI screenshots pretending to be working interfaces;
- “coming soon” without a useful next action;
- style decisions chosen because they are easy for an AI to generate.

If a design choice feels like a default model habit, challenge it and choose the cleaner, more specific option.

## 15. DESIGN DIALS

Set and record these before visual work:

```json
{
  "design_variance": 1,
  "motion_intensity": 1,
  "visual_density": 1,
  "editorial_to_utilitarian": 1,
  "warm_to_cool": 1,
  "organic_to_geometric": 1,
  "quiet_to_expressive": 1
}
```

Use 1-10 scales. The project truth determines the values.

Do not default automatically to high variance or heavy motion. Motion begins only after structure, accessibility, and mobile behavior are stable.
